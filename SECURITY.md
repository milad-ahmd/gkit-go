# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.x (latest) | ✅ |
| < 1.0 | ❌ |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately via **GitHub's Security Advisory** feature:

1. Go to the [Security tab](https://github.com/miladhzz/gkit/security/advisories) of this repository
2. Click **"Report a vulnerability"**
3. Fill in the details: affected package(s), Go version, reproduction steps, and impact

Alternatively, email **miladahmadi803@gmail.com** with the subject line:
```
[SECURITY] gkit: <short description>
```

## What to Include

- Affected `pkg/` package(s) and version range
- Go version and OS
- Minimal reproduction case
- Potential impact (data leak, panic, privilege escalation, etc.)

## Disclosure Timeline

| Step | Target time |
|---|---|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 7 days |
| Patch & advisory | Within 30 days |
| Public disclosure | After patch is released |

We follow responsible disclosure. We will credit reporters in the release notes unless you prefer to remain anonymous.
